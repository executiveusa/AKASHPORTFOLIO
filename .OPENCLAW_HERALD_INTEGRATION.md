## Architecture

```
┌─────────────────────────────────────┐
│      OpenClaw Gateway               │
│  (Multi-channel message ingestion)  │
└──────────────────┬──────────────────┘
                   │
         POST /api/herald/message
         {session_id, message, context}
                   │
                   ▼
┌─────────────────────────────────────┐
│      HERALD Integration Layer       │
│  (Intent classification + routing)  │
└──────────────────┬──────────────────┘
                   │
         semanticRoute(intent)
                   │
                   ▼
┌─────────────────────────────────────┐
│      HERALD Tool Router             │
│  (pgvector semantic search)         │
└──────────────────┬──────────────────┘
                   │
         SELECT best_tool
         WHERE similarity > 0.7
                   │
                   ▼
┌─────────────────────────────────────┐
│      Skill Execution                │
│  (Agent + Tool invocation)          │
└──────────────────┬──────────────────┘
                   │
              Tool output
                   │
                   ▼
┌─────────────────────────────────────┐
│      Response Formatting            │
│  (Channel-specific output)          │
└─────────────────────────────────────┘
```

## Intent Classifications

Supported intents in HERALD (expanded per Tablet III):

### Tier 1: Primary Intents
- `lead_qualification` — Prospect assessment, company research, fit analysis
- `information_request` — Questions about services, FAQ, general inquiries
- `schedule_meeting` — Calendar booking, call scheduling, appointment requests
- `project_inquiry` — Design/development project interest, RFP, portfolio review
- `support_request` — Technical support, bug reports, help needed

### Tier 2: Secondary Intents
- `sales_proposal` — Request for pricing, custom quote, proposal generation
- `content_request` — Blog post access, resource download, media request
- `feedback_submission` — Testimonial, review, bug report, feature request
- `team_collaboration` — Internal team message, workflow request
- `escalation_required` — Cannot handle intent, human intervention needed

### Tier 3: Special Intents
- `general_inquiry` — Catch-all, ambiguous intent
- `spam_detection` — Likely spam or malicious intent
- `error` — System error occurred during processing

## Intent Classification Logic

Current implementation uses heuristic-based classification (Tablet VIII decay-ready):

```typescript
// Keywords mapped to intents
"lead|prospect|company|opportunity" → lead_qualification (0.85)
"what|how|why|explain|info" → information_request (0.80)
"schedule|meeting|call|book|date" → schedule_meeting (0.90)
"project|portfolio|work|design" → project_inquiry (0.82)
"help|support|issue|bug|error" → support_request (0.88)
```

**Future**: Replace with semantic similarity + LLM classification using:
- Text embedding (text-embedding-3-small via LiteeLM)
- Intent vector database (pgvector in Supabase)
- Confidence decay (5%/day per Tablet VIII)

## Vibe Node Updates

After skill execution, update confidence scores per Tablet VIII:

**Request**:
```json
POST /api/herald/vibe-node/update
{
  "agent": "Scout",
  "skill": "lead_scoring",
  "confidence": 0.92,
  "timestamp": 1692345600000
}
```

**Decay Schedule**:
- Weekly cron job (Saturdays 00:00 UTC)
- Decay factor: 0.95 (5% per day)
- Minimum threshold: 0.60 (remove edges below this)
- Formula: `new_confidence = old_confidence * (0.95 ^ days_since_update)`

Example: Skill at 0.80 confidence after 7 days = 0.80 * 0.95^7 = 0.56 (removed)

## Error Handling

### Message Processing Errors

| Error | Status | Response |
|-------|--------|----------|
| Invalid intent | 400 | `{ error: "Unable to classify intent" }` |
| No matching tool | 404 | `{ error: "No suitable tool found" }` |
| Tool execution failure | 500 | `{ error: "Skill execution failed" }` |
| HERALD unavailable | 503 | `{ error: "Routing service unavailable" }` |
| Timeout (>30s) | 504 | `{ error: "Request timeout" }` |

### Fallback Behavior

When no tool matches confidence threshold (0.70):
1. Log classification failure
2. Offer escalation message to user
3. Queue for human review
4. Response: "I'll escalate this to our team member..."
