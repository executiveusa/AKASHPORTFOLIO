| Component | Spec | Details |
|-----------|------|---------|
| Compute | 1x 2 CPU, 4GB RAM | Single node deployment |
| Storage | 20GB SSD | Session database + logs |
| Database | PostgreSQL 14+ | Shared managed instance |
| Cache | Redis 6+ | Managed cache service |

| Component | Spec | Details |
|-----------|------|---------|

| Monitoring | Full stack | Prometheus, Datadog, or New Relic |
| Logging | Centralized | ELK stack, Loki, or Splunk |

## Deployment Steps

### 1. Pre-Deployment Checklist

```bash
# Requirements
- [ ] Docker installed (20.10+)
- [ ] kubectl configured (1.24+)
- [ ] Secrets manager setup (Vault, AWS Secrets Manager)
- [ ] Database backups configured
- [ ] Monitoring system ready
- [ ] SSL certificates issued
- [ ] API keys for all channels obtained
```

### 2. Build & Push Docker Image

```bash
# Build multi-stage image
docker build -t kupuri/openclaw-gateway:latest \
  -f apps/openclaw-gateway/docker/Dockerfile \
  apps/openclaw-gateway/

# Tag for registry
docker tag kupuri/openclaw-gateway:latest \
  registry.example.com/kupuri/openclaw-gateway:1.0.0

# Push to registry
docker push registry.example.com/kupuri/openclaw-gateway:1.0.0
```

### 3. Deploy to Kubernetes

```bash
# Apply secrets
kubectl create secret generic openclaw-secrets \
  --from-env-file=.env.production \
  -n kupuri

# Apply manifests
kubectl apply -f k8s/openclaw-gateway.yaml -n kupuri
kubectl apply -f k8s/herald.yaml -n kupuri

# Verify deployment
kubectl rollout status deployment/openclaw-gateway -n kupuri
kubectl get pods -n kupuri
```

### 4. Verify Deployment

```bash
# Check gateway health
kubectl port-forward svc/openclaw-gateway 3000:3000 &
curl http://localhost:3000/health

# Check HERALD health
kubectl port-forward svc/herald 4200:4200 &
curl http://localhost:4200/health

# View logs
kubectl logs -f deployment/openclaw-gateway -n kupuri
```

### Deployment

```yaml
# k8s/openclaw-gateway.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: openclaw-gateway
  namespace: kupuri
spec:
  replicas: 3
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
  selector:
    matchLabels:
      app: openclaw-gateway
  template:
    metadata:
      labels:
        app: openclaw-gateway
      annotations:
        prometheus.io/scrape: "true"
        prometheus.io/port: "3000"
    spec:
      serviceAccountName: openclaw
      containers:
      - name: gateway
        image: registry.example.com/kupuri/openclaw-gateway:1.0.0
        imagePullPolicy: Always
        ports:
        - name: http
          containerPort: 3000
          protocol: TCP
        envFrom:
        - configMapRef:
            name: openclaw-config
        - secretRef:
            name: openclaw-secrets
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "1Gi"
            cpu: "1000m"
        livenessProbe:
          httpGet:
            path: /health
            port: http
          initialDelaySeconds: 10
          periodSeconds: 10
          timeoutSeconds: 5
          failureThreshold: 3
        readinessProbe:
          httpGet:
            path: /health
            port: http
          initialDelaySeconds: 5
          periodSeconds: 5
          timeoutSeconds: 3
          failureThreshold: 2
        volumeMounts:
        - name: data
          mountPath: /data
      volumes:
      - name: data
        persistentVolumeClaim:
          claimName: openclaw-data
      affinity:
        podAntiAffinity:
          requiredDuringSchedulingIgnoredDuringExecution:
          - labelSelector:
              matchExpressions:
              - key: app
                operator: In
                values:
                - openclaw-gateway
            topologyKey: kubernetes.io/hostname
```

## Database Setup

### PostgreSQL with pgvector

```sql
-- Create vector extension
CREATE EXTENSION IF NOT EXISTS vector;

-- Create sessions table
CREATE TABLE sessions (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  channel TEXT NOT NULL,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL,
  context JSONB NOT NULL,
  metadata JSONB,
  UNIQUE(user_id, channel)
);

-- Create messages table
CREATE TABLE messages (
  id TEXT PRIMARY KEY,
  session_id TEXT NOT NULL,
  role TEXT NOT NULL,
  content TEXT NOT NULL,
  intent TEXT,
  timestamp INTEGER NOT NULL,
  FOREIGN KEY(session_id) REFERENCES sessions(id)
);

-- Create indices
CREATE INDEX idx_session_user_channel ON sessions(user_id, channel);
CREATE INDEX idx_message_session ON messages(session_id);
CREATE INDEX idx_message_timestamp ON messages(timestamp DESC);

-- Vibe node embeddings table
CREATE TABLE vibe_node_embeddings (
  id TEXT PRIMARY KEY,
  agent TEXT NOT NULL,
  skill TEXT NOT NULL,
  embedding vector(1536),
  confidence REAL NOT NULL,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(agent, skill)
);

-- Create vector index
CREATE INDEX idx_vibe_embedding ON vibe_node_embeddings USING ivfflat (embedding vector_cosine_ops);
```

### Backup Strategy

```bash
# Daily backups
00 2 * * * pg_dump -U postgres kupuri_herald | gzip > /backups/herald-$(date +\%Y\%m\%d).sql.gz

# Retention policy: 30 days
find /backups -name "herald-*.sql.gz" -mtime +30 -delete

# Test restore monthly
pg_restore -U postgres -d kupuri_herald_test /backups/herald-latest.sql.gz
```

## Scaling Considerations

### Horizontal Scaling

- Gateway is stateless (sessions in DB, not memory)
- Min 3 replicas for HA
- Max 10 replicas recommended (cost vs performance)
- HPA triggers at 70% CPU or 80% memory

### Vertical Scaling

- Per-pod: 4 CPU, 8GB RAM maximum
- Use cluster autoscaling for node expansion
- Monitor database connection pool exhaustion

### Rate Limiting

```
# Per IP
100 requests/minute

# Per session
10 messages/minute

# Per channel
1000 messages/minute
```
