## Integration Tests

### Message Flow Tests

**Test**: Complete user message → HERALD → response

```bash
npm test -- __tests__/integration.test.ts -t "should process complete message flow"
```

**Flow**:
```
1. User sends WhatsApp message: "Find leads in AI space"
   └─ EventBridge.sendMessageToHerald()
      └─ HERALD /api/herald/message
         └─ classifyIntentFromMessage() → "lead_qualification"
         └─ semanticRoute() → Scout skill
         └─ Execute Scout
         └─ Return response
   └─ ResponseRouter.formatForChannel('whatsapp', response)
   └─ WhatsAppAdapter.sendMessage(response)
```

### Error Handling Tests

**Test**: Various failure scenarios

```bash
npm test -- __tests__/integration.test.ts -t "should handle.*gracefully"
```

**Scenarios**:
- HERALD timeout (>30s)
- No matching tool found
- Tool execution failure
- Invalid message format
- Channel not enabled
- Session storage error

### Multi-Channel Tests

**Test**: Same message across different channels

```bash
npm test -- __tests__/integration.test.ts -t "should format.*for"
```

**Channels**:
- WhatsApp (Twilio emojis + quick replies)
- Telegram (Markdown + inline keyboards)
- Slack (blocks + thread support)
- Discord (embeds + components)

## E2E Tests

### Local Testing (No API Keys)

```bash
# 1. Start OpenClaw Gateway (mock mode)
npm run dev

# 2. In another terminal, run health check
curl http://localhost:3000/health

# 3. Test session creation
curl -X POST http://localhost:3000/api/gateway/session \
  -d '{"user_id": "test-123", "channel": "whatsapp"}' \
  -H "Content-Type: application/json"
```

### Integration Testing (With API Keys)

```bash
# 1. Configure .env with real credentials
cp .env.example .env
# Edit .env with actual API keys

# 2. Enable channels in .env
WHATSAPP_ENABLED=true
TELEGRAM_ENABLED=true
SLACK_ENABLED=true
DISCORD_ENABLED=true

# 3. Start gateway
npm run dev

# 4. Send test message (e.g., WhatsApp)
curl -X POST http://localhost:3000/api/gateway/send \
  -d '{
    "channel": "whatsapp",
    "sender_id": "+1234567890",
    "text": "Find AI recruitment startups"
  }' \
  -H "Content-Type: application/json"

# 5. Verify response via real channel
```

### Production Testing

```bash
# 1. Build for production
npm run build

# 2. Run on production server
NODE_ENV=production npm start

# 3. Monitor logs
tail -f logs/gateway.log

# 4. Health check
curl https://your-domain.com/health
```

### Enable Debug Logging

```bash
LOG_LEVEL=debug npm run dev
```

### Common Issues

**Issue**: "HERALD health check failed"
- Check: `curl http://localhost:4200/health`
- Fix: Start HERALD on port 4200

**Issue**: "WhatsApp messages not received"
- Check: Webhook URL in Twilio console
- Fix: Ensure public HTTPS URL configured

**Issue**: "Session database locked"
- Check: Multiple instances running?
- Fix: Kill other processes, use `:memory:` for testing

**Issue**: "Intent classification timeout"
- Check: LiteeLM embedding service
- Fix: Start LiteeLM on port 4000
