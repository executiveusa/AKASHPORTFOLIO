## Staging Deployment (Days 11-12)

### Deploy to Staging

```bash
# 1. Build and push image
docker build -t registry.example.com/kupuri/openclaw-gateway:1.0.0 .
docker push registry.example.com/kupuri/openclaw-gateway:1.0.0

# 2. Apply to staging cluster
kubectl apply -f k8s/ -n kupuri-staging

# 3. Verify deployment
kubectl rollout status deployment/openclaw-gateway -n kupuri-staging

# 4. Run smoke tests
npm run test:smoke
```

## Rollback Plan (If Needed)

### Trigger Conditions

Rollback initiated if:
- Error rate >5% (15+ min sustained)
- Latency P95 >10s (15+ min sustained)
- Database corruption detected
- Critical security vulnerability found
- HERALD unreachable (>5 min)

### Rollback Steps

```bash
# 1. Alert team
kubectl rollout undo deployment/openclaw-gateway -n kupuri

# 2. Verify rollback
kubectl rollout status deployment/openclaw-gateway -n kupuri

# 3. Run smoke tests
npm run test:smoke

# 4. Verify metrics
# Confirm error rate and latency returned to normal

# 5. Incident post-mortem
# Schedule team debrief within 24 hours
```

### Expected Rollback Time: <5 minutes
